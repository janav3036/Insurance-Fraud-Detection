Model

To identify potentially fraudulent claims, a supervised machine learning model was trained using the engineered dataset. The chosen algorithm for this project is XG Boost (Extreme Gradient Boosting), a powerful ensemble learning method used widely for structured data problems.

Why XGBoost?

  • Excellent performance on tabular dataset
  • Handles non linear relationships between features
  • Built-in regularization to reduce overfitting
  • Works well with imbalanced datasets

Handling Class Imbalance

Insurance fraud datasets are typically imbalanced because fraudulent claims are much rarer than legitimate ones. To address this issue, the model was trained using the scale_pos_weight parameter to balance fraud and non-fraud observations.

Additionally, the classification threshold was adjusted to prioritize recall, ensuring that fraudulent claims are more likely to be detected even at the cost of some false positives.

Model Performance

The model was evaluated using several metrics including accuracy, precision, recall, and ROC-AUC.

  • ROC-AUC Score: ~0.63
  • Fraud Recall: ~0.75+
  • Accuracy: ~0.48

In fraud detection, recall is often prioritized because missing fraudulent claims can be more costly than incorrectly flagging legitimate ones.
